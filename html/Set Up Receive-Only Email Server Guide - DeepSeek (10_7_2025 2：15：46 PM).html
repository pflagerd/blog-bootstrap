Good! Postfix is installed, but I notice a few things from your installation output that need to be adjusted for your receive-only setup. The automatic configuration set some values that might not be ideal.

Current Issues to Fix:

1. Hostname Issue

The installer set:

text
myhostname: srv447782.hstgr.cloud

But you probably want your domain (pflager.com) as the mail host.

2. Destination Domains

It set destinations to multiple domains including the server hostname, but you likely only want your main domain.

Let's Fix the Configuration:

Step 1: Edit Postfix Main Configuration

bash
sudo nano /etc/postfix/main.cf

Step 2: Update with Receive-Only Configuration

Replace or modify these lines:

ini
# Basic settings
myhostname = mail.pflager.com
mydomain = pflager.com
myorigin = $mydomain

# Only receive for your domain
mydestination = $myhostname, localhost.$mydomain, localhost, $mydomain

# Network settings (IPv4 only for simplicity)
inet_interfaces = all
inet_protocols = ipv4

# Security - restrict relaying
mynetworks = 127.0.0.0/8 [::ffff:127.0.0.0]/104 [::1]/128
relay_domains = 
smtpd_recipient_restrictions = permit_mynetworks, reject_unauth_destination

# Mailbox settings
home_mailbox = Maildir/

# Prevent outgoing mail
default_transport = error:No outgoing mail
relay_transport = error:No outgoing mail

Step 3: Update /etc/mailname (if needed)

bash
sudo nano /etc/mailname

Ensure it contains:

text
pflager.com

Step 4: Reload Postfix

bash
sudo systemctl reload postfix
sudo systemctl status postfix

Step 5: Install Dovecot for Mail Access

bash
sudo apt install dovecot-imapd dovecot-pop3d

Step 6: Configure Dovecot

Edit the main Dovecot config:

bash
sudo nano /etc/dovecot/dovecot.conf

Ensure it includes:

ini
protocols = imap pop3
mail_location = maildir:~/Maildir

Configure authentication:

bash
sudo nano /etc/dovecot/conf.d/10-auth.conf
ini
disable_plaintext_auth = no
auth_mechanisms = plain login

Step 7: Create a Test User

bash
sudo adduser testuser
# Follow prompts to set password

Step 8: Configure DNS Records in GoDaddy

Now set up your DNS records:

MX Record:

text
Type: MX
Name: @
Value: mail.pflager.com
Priority: 10
TTL: 3600 (1 hour)

A Record for mail subdomain:

text
Type: A
Name: mail
Value: [Your VPS IP address]
TTL: 3600

Step 9: Test Your Configuration

Check Postfix config:

bash
sudo postconf -n

Test locally first:

bash

echo "Test email" | mail -s "Test" testuser@pflager.com

Step 10: Firewall Configuration

bash
sudo ufw allow 25/tcp    # SMTP
sudo ufw allow 110/tcp   # POP3
sudo ufw allow 143/tcp   # IMAP
sudo ufw allow 993/tcp   # IMAPS
sudo ufw allow 995/tcp   # POP3S

Verification Steps

  1. Check services are running:

    bash
    sudo systemctl status postfix dovecot

  2. Test from external email:

    • Wait 1-2 hours for DNS propagation

    • Check logs: sudo tail -f /var/log/mail.log

  3. Verify DNS:

    bash
    dig MX pflager.com

Important Notes:

  • The automatic installation used your server's hostname, but for a proper mail server, you want mail.yourdomain.com

  • DNS changes can take 1-24 hours to propagate globally

  • Make sure your VPS provider doesn't block port 25 (some do for new accounts)
