1.5 — One Away

Equivalent problems on LeetCode and GeeksForGeeks

The Problem

There are three types of edits that can be performed on strings: insert a character, remove a character, or replace a character. Given two strings, write a function to check if they are one edit (or zero edits) away.

Examples:  pale, ple → true  ·  pales, pale → true  ·  pale, bale → true  ·  pale, bake → false

LeetCode Equivalents

#161 One Edit Distance Premium

Given two strings s and t, return true if they are both one edit distance apart — by inserting, deleting, or replacing exactly one character. This is essentially the same problem as CtCI 1.5.

⚠ Subtle difference: LC 161 requires exactly one edit (identical strings return false), whereas CtCI 1.5 allows zero edits (identical strings return true).
leetcode.com/problems/one-edit-distance →
#72 Edit Distance Hard · Free

The more general parent problem: given two strings, return the minimum number of operations (insert, delete, replace) to convert one into the other. Solved with dynamic programming (Levenshtein distance). You could use it to solve CtCI 1.5 by checking if the result is ≤ 1, but the intended CtCI approach is a simpler O(n) single-pass — DP is overkill here. Still excellent background context.
